다음 정적 웹사이트 코드베이스를 레이아웃만 정리해줘.

[중요 전제]
- 계산 기능, 산식, 업로드 처리, 엑셀/PDF 처리, JS 변수/함수/이벤트 로직은 절대 변경하지 말 것.
- 이번 작업은 레이아웃만 수정한다. 즉 폭, 여백, 카드 간격, 버튼 줄 정렬, 테이블 스크롤, 모바일 배치만 손본다.
- 정적 페이지 + 자바스크립트만으로 기능 구현하는 사이트다. 별도 요청 없으면 다른 언어로 구현하지 말 것.
- 기존 전체 공통 /static/style.css의 UI 톤을 기준으로 맞출 것.
- 다크모드 금지.
- 파일 업로드 기능이 있는 페이지는 항상 드래그 앤 드롭 업로드 동작을 유지할 것.
- JS 훅으로 쓰일 수 있는 id, name, data-*, class는 함부로 바꾸지 말 것.
- 결과 코드는 파일 다운로드가 아니라, 복사 붙여넣기 가능하도록 전체 코드를 텍스트로 줄 것.
- 수정사항은 코드가 어떻게 바뀌었는지 5줄 설명과 함께 줄 것.
- 왜 그렇게 동작하는지 10줄 요약해서 알려줄 것.
- 답변이 모호할 때는 2~3개의 옵션을 먼저 제시한 뒤, 기본안 1개를 선택해서 진행하고 질문으로 멈추지 말 것.

[목표 구조]
1. 홈/문서형 페이지는 기존 .container를 유지한다.
2. 넓은 계산기/도구 페이지는 .container에 의존하지 않고 전용 main 클래스를 쓴다.
3. /static/style.css에 아래 계열을 추가한다.
   - .page-main
   - .page-main--compact
   - .page-main--slim
   - .page-main--narrow
   - .page-main--medium
   - .page-main--wide
   - .page-main--xwide
   - .page-main--xxwide
   - .page-main--full
4. 각 index.html의 인라인 CSS에서 wrapper 폭/패딩 관련 규칙만 제거한다.
   - main.wrap { ... }
   - .wrap { max-width: ... }
   - .container { max-width: ... }
   - main { max-width: ... }
5. 그리드, 카드, 테이블, 입력칸, 계산 로직, 업로드 로직, 버튼 동작은 그대로 둔다.

1. 먼저 2~3개 정리 방식 옵션을 짧게 제시

3. 수정한 파일마다 전체 코드 전문을 복붙 가능하게 제공

5. 마지막에 전체 동작 원리 10줄 요약 제공
