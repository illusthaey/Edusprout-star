당신은 프론트엔드 개발자입니다.

아래의 기존 단일 HTML 계산기를 기준으로, **출입문개폐전담원 월별 인건비 계산기**를
**오프라인 단일 HTML 파일 1개**로 다시 수정해 주세요.

## 최종 목표
사용자가 브라우저에서 HTML 파일 하나만 열어도 아래 기능이 전부 동작해야 합니다.

- 출입문개폐전담원 월별 인건비 계산
- 엑셀처럼 방향키로 셀 이동
- 날짜는 달력 선택 + 직접 타이핑 둘 다 가능
- 근무시간도 키보드로 바로 입력 가능
- 계산 결과를 **실제 `.xlsx` 파일**로 다운로드
- 엑셀 결과물은 **업로드한 서식과 매우 유사한 2개 시트**로 생성

---

## 반드시 반영할 수정사항

### 1. 기본값
- 시급 기본값: **12,380원**
- 생년월일 기본값: **1949-08-10**
- 최초임용일 기본값: **2021-03-01**
- 직종 기본값: `출입문개폐전담원`

### 2. 불필요 기능 제거
아래 기능은 제거해 주세요.
- 업로드용 엑셀 서식 다운로드
- 엑셀 파일 불러오기
- 상단/기본정보 섹션 쪽의 계산 결과 엑셀 다운로드 버튼
- 업로드 관련 안내 문구

즉, **엑셀 다운로드 버튼은 4번 산출내역 섹션에만** 있으면 됩니다.

### 3. 날짜 입력 UX
다음 필드는 모두 **달력으로 선택 가능 + 직접 타자 입력 가능**해야 합니다.
- 지급월
- 지급일자
- 생년월일
- 최초임용일

직접 입력할 때 아래 형식을 허용해 주세요.
- 지급월: `202603`, `2026-03`
- 날짜: `19490810`, `1949-08-10`, `1949.08.10`

입력 후 자동으로 표준 형식으로 정규화되게 해 주세요.
- 지급월 → `YYYY-MM`
- 날짜 → `YYYY-MM-DD`

특히 생년월일은 고령 사용자가 많으므로 **직접 입력이 매우 편해야 합니다.**

### 4. 근무시간 입력 UX
요일별 근무시간은 `input type="time"`만 쓰지 말고,
**키보드 타이핑이 쉬운 텍스트 입력 방식**으로 바꿔 주세요.

예를 들어 아래처럼 입력 가능하게 해 주세요.
- `0710`
- `710`
- `07:10`

입력 후 자동으로 `HH:MM` 형식으로 정규화되게 해 주세요.

### 5. 엑셀 같은 셀 이동
근무시간표와 달력 입력 영역에서는
**키보드 방향키 / Enter로 엑셀처럼 셀 이동**이 가능해야 합니다.

적용 대상 예시:
- 요일별 근무시간 입력 셀
- 달력의 출근 토글
- 일별 수동 근로시간 입력 칸

### 6. 결과 엑셀(.xlsx) 형식
다운로드되는 결과 파일은 반드시 **실제 `.xlsx` 형식**이어야 합니다.

그리고 시트는 **2개만** 생성해 주세요.
1. `출근내역`
2. `임금명세서`

### 7. 엑셀 서식 일치 요구
사용자가 업로드한 `출입문개폐전담원_인건비_202603.xlsx`를 기준으로,
다운로드되는 엑셀 파일의 레이아웃과 스타일을 최대한 유사하게 구현해 주세요.

반드시 맞춰야 할 요소:
- 모든 셀 **가운데 정렬**
- 제목 폰트 크기 느낌
- 회색 배경 셀 색상 느낌
- 셀 테두리
- 열 너비
- 행 높이
- 병합 구조
- 시트 구성

가장 좋은 방식은:
- 업로드한 `.xlsx`를 **내장 템플릿(base64 등)** 으로 HTML 안에 포함하고
- 저장 시 값만 바꿔서 다시 다운로드하는 방식입니다.

즉,
**스타일을 새로 대충 만드는 방식보다 템플릿 기반으로 정확하게 맞추는 방식**을 우선 사용해 주세요.

### 8. 시트 1 (출근내역) 구조
다운로드 결과는 첨부 이미지처럼 보여야 합니다.

핵심 구조:
- 제목: `2026년 3월 출입문개폐요원 출근 내역` 형식
- 달력형 날짜/출근표
- 우측에 `요일별 근무 시간`
- 하단 요약표
  - 근무일수
  - 적용 시급
  - 유급 근로시간
  - 보수 지급액

### 9. 시트 2 (임금명세서) 구조
다운로드 결과는 첨부 이미지처럼 보여야 합니다.

핵심 구조:
- 제목: `2026년 3월 임금명세서`
- 상단 기본정보
  - 소속
  - 지급일
  - 성명
  - 직종
  - 생년월일
  - 최초임용일
- 중간 급여내역 / 공제내역
- 하단
  - 급여총액 계(A)
  - 공제액 계(B)
  - 실수령액(A-B)

### 10. 웹페이지 미리보기 UI도 엑셀 느낌으로
웹페이지의 4번 산출내역 섹션 미리보기도
업로드한 엑셀과 유사한 레이아웃 느낌이 나게 해 주세요.

즉, HTML 미리보기에서도 다음을 반영해 주세요.
- 회색 배경 제목줄
- 가운데 정렬
- 표 테두리
- 엑셀 같은 셀 모양
- 시트 1 / 시트 2가 실제 저장 결과와 비슷하게 보이도록

### 11. 유지 기능
가능하면 아래 기능은 유지해 주세요.
- localStorage 자동 저장
- 건강보험 / 장기요양보험 / 고용보험 자동 계산
- 소득세 / 지방소득세 / 국민연금 / 기타 공제 수기입력
- 근무시간 자동 적용
- 달력 출근 토글
- 일별 근로시간 수동입력 옵션
- PDF는 외부 라이브러리 없이 브라우저 인쇄 방식이어도 됨

### 12. 오프라인 단일 HTML
최종 결과물은 반드시 아래 조건을 만족해야 합니다.
- HTML 1파일
- CSS는 `<style>` 안에 모두 포함
- JS는 `<script>` 안에 모두 포함
- 외부 CDN 사용 금지
- `/static/...` 의존성 제거
- 인터넷 없이 동작
- `.xlsx` 다운로드도 오프라인에서 동작

## 권장 구현 방식
- JSZip 소스를 HTML 안에 직접 내장
- 업로드된 `출입문개폐전담원_인건비_202603.xlsx`를 base64 템플릿으로 내장
- `xl/worksheets/sheet1.xml`, `sheet2.xml` 값만 바꿔서 저장
- 기존 스타일, 병합, 열너비, 행높이는 템플릿 그대로 유지
